# Analyze-Session Skill Vision

The Analyze-Session skill is live and transforming every Claude Code project. I run /analyze-session latest, and it instantly delivers a crystal-clear Markdown report right in my workspace.

Each report starts with a snapshot: total tokens spent, dollar cost broken down by model and cache hits, session duration, lines of code changed, and a satisfaction score from Claude's own analysis. Tables make it scannable— I see exactly where tokens vanished or tools failed.

The 'What Happened' section gives a one-paragraph story of the session's flow, pulling from Claude's facet summary. Then, token breakdowns reveal the truth: 60% on cache reads? Perfect efficiency. 40% on repeated prompts? Time to tighten.

Tool usage shines: top performers, error-prone ones flagged with counts and categories. Efficiency metrics show lines per 1K tokens—my MCP builds are lean machines now.

Issues pop bold: 'High friction on buggy_code (3x)—add validation skill.' Fixes follow conversational: 'Shorten this prompt loop, here's the before/after. Expected 25% token save.'

For projects, it aggregates trends: cost dropping over 10 sessions, success rate climbing to 95%, common pitfalls ranked. Next actions checklist: 'Create retry skill for MCP timeouts. Commit refined CLAUDE.md.'

This skill closes the loop. Every session teaches the next. My agents evolve fast—accurate, light, unstoppable. /analyze-session is my secret weapon.
